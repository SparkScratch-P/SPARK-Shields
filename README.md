function openCity(evt, cityName) {

# GitHub Stats Card

Copy-paste this into your markdown content, and that's it. Simple!

Change the `?username=` value to your GitHub's username.

```md
[![GitHub stats](https://github-readme-stats.vercel.app/api?username=SparkScratch-P)](https://github.com/SparkScratch-P/github-readme-stats)
```

_Note: Available ranks are S+ (top 1%), S (top 25%), A++ (top 45%), A+ (top 60%), and B+ (everyone).
The values are calculated by using the [cumulative distribution function](https://en.wikipedia.org/wiki/Cumulative_distribution_function) using commits, contributions, issues, stars, pull requests, followers, and owned repositories.
The implementation can be investigated at [src/calculateRank.js](./src/calculateRank.js)_

<h3 id="hiding-individual-stats">Hiding individual stats</h3>
<p>To hide any specific stats, you can pass a query parameter <code>?hide=</code> with comma-separated values.</p>
<blockquote>
<p>Options: <code>&amp;hide=stars,commits,prs,issues,contribs</code></p>
</blockquote>
<pre><code class="lang-md">![<span class="hljs-string">GitHub stats</span>](<span class="hljs-link">https://github-readme-stats.vercel.app/api?username=SparkScratch-P&amp;hide=contribs,prs</span>)
</code></pre>
<h3 id="adding-private-contributions-count-to-total-commits-count">Adding private contributions count to total commits count</h3>
<p>You can add the count of all your private contributions to the total commits count by using the query parameter <code>?count_private=true</code>.</p>
<p><em>Note: If you are deploying this project yourself, the private contributions will be counted by default. Otherwise, you need to choose to share your private contribution counts.</em></p>
<blockquote>
<p>Options: <code>&amp;count_private=true</code></p>
</blockquote>
<pre><code class="lang-md">![<span class="hljs-string">GitHub stats</span>](<span class="hljs-link">https://github-readme-stats.vercel.app/api?username=SparkScratch-P&amp;count_private=true</span>)
</code></pre>

<h3 id="customization">Customization</h3>
<p>You can customize the appearance of your <code>Stats Card</code> or <code>Repo Card</code> however you wish with URL params.</p>
<h4 id="common-options-">Common Options:</h4>
<ul>
<li><code>title_color</code> - Card&#39;s title color <em>(hex color)</em></li>
<li><code>text_color</code> - Body text color <em>(hex color)</em></li>
<li><code>icon_color</code> - Icons color if available <em>(hex color)</em></li>
<li><code>border_color</code> - Card&#39;s border color <em>(hex color)</em>. (Does not apply when <code>hide_border</code> is enabled)</li>
<li><code>bg_color</code> - Card&#39;s background color <em>(hex color)</em> <strong>or</strong> a gradient in the form of <em>angle,start,end</em></li>
<li><code>hide_border</code> - Hides the card&#39;s border <em>(boolean)</em></li>
<li><code>theme</code> - name of the theme, choose from <a href="./themes/README.md">all available themes</a></li>
<li><code>cache_seconds</code> - set the cache header manually <em>(min: 1800, max: 86400)</em></li>
<li><code>locale</code> - set the language in the card <em>(e.g. cn, de, es, etc.)</em></li>
<li><code>border_radius</code> - Corner rounding on the card_</li>
</ul>
<h5 id="gradient-in-bg_color">Gradient in bg_color</h5>
<p>You can provide multiple comma-separated values in bg_color option to render a gradient, the format of the gradient is :-</p>
<pre><code>&amp;bg_color=DEG,COLOR1,COLOR2,COLOR3...COLOR10
</code></pre><blockquote>
<p>Note on cache: Repo cards have a default cache of 4 hours (14400 seconds) if the fork count &amp; star count is less than 1k, otherwise, it&#39;s 2 hours (7200 seconds). Also, note that the cache is clamped to a minimum of 2 hours and a maximum of 24 hours.</p>
</blockquote>

</ul>
